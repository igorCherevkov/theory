Жёсткая ссылка — это дополнительное имя для существующего файла. Оба имени указывают на один и тот же инод (inode) в файловой системе.
Символьная ссылка — это отдельный файл, который указывает на другой файл или директорию по пути.

Жёсткие ссылки могут быть созданы только в пределах одной файловой системы. Нельзя создать жёсткую ссылку на файл, находящийся на другом диске или разделе.
Символьная ссылка может указывать на файл или директорию, находящиеся в любом месте файловой системы.

Жёсткие ссылки непосредственно указывают на содержимое файла, минуя дополнительный уровень косвенности.
При доступе к символьной ссылке система сначала переходит по указанному пути и открывает целевой файл.

Файл остаётся доступным по любой из жёстких ссылок до тех пор, пока не будут удалены все ссылки. Удаление одного имени не влияет на доступность файла по другим именам.
Если целевой файл удалён, символьная ссылка остаётся, но становится "битой" (нерабочей) ссылкой.

Жёсткие ссылки неотличимы от обычных файлов, так как они указывают на один и тот же инод. В выводе команды ls -l вы можете увидеть количество ссылок в поле ссылок (колонка после прав доступа).
Символьные ссылки часто отображаются с стрелкой (->), указывающей на целевой файл, и их можно распознать по специальному атрибуту при выводе команды ls -l.

20. Как посмотреть список процессов, если известна часть имени процесса 

pgrep -f <часть названия процесса> -f: Этот флаг указывает pgrep искать по всему аргументу командной строки, а не только по имени процесса.
ps aux | grep <часть названия процесса>
ps Утилита для отображения информации о процессах (prosess status)
aux: Опции команды ps, где a означает вывод процессов для всех пользователей, u показывает детализированную информацию о процессе, а x выводит также процессы, не привязанные к терминалу.
| (pipe): Этот символ используется для передачи вывода одной команды на вход другой. В данном случае, вывод ps aux будет передан на вход команды grep.
grep: Эта утилита используется для фильтрации текста с помощью регулярных выражений. В контексте ps aux | grep, grep фильтрует вывод ps aux и выводит только строки, которые содержат заданную строку или регулярное выражение.
